webpack基础知识记录

# 提问：webpack的构建流程

解析：我境界不足不知道他考点是什么（因为我还是个初级前端菜鸟），可能就是考验你懂不懂这些，在实际中或许会有需要用到的地方

答案1：

webpack启动之后会读取命令行的配置项和配置文件的配置项，接着从entry开始，递归解析entry所依赖的module，根据配置找到module对应的loader进行相应的处理转换，对module转换后则进行解析，解析的成果是一个个chunk，最后将所有chunk转换成文件输出的output

在整个构建过程中，webpack会在恰当的时机执行plugin。

答案2：

每种答案都可以，就看能理解哪个就说哪一个。

  1. 检查是否有安装官方脚手架，如没有则提示安装，如果不止一个脚手架则提示要删除一个
  2. 初始化参数，从命令行和配置文件读取和合并参数得出最终参数（命令行优先）。
  3. 初始化compiler对象，加载所有配置的插件，将他们挂载到comiler对象上
  4. 根据配置的entry找出所有入口文件。
  5. 从入口文件出发，对文件中的模块使用相应的loader进行解析（翻译），然后再找出该模块依赖的模块加入到依赖数组，如此递归直到所有入口依赖文件都经过处理。
  6. 完成模块编译工作，我们得知了每个模块所持有的内容以及依赖关系。
  7. 根据入口和模块之间的依赖关系，组成一个个包含多个模块的chunk，再把每个chunk转换成单独的文件加入输出列表。
  8. 得到输出列表就等于决定了输出内容，这时候再根据配置文件的路径和文件名把内容写进去就好。
  9. webpack会在以上过程中再特定时间发出广播事件去监听需要用到插件的地方进行功能拓展，并调用webpack提供的api改变webpack的运行结果

# 提问：webpack常用的loader和plugin

解析：考验你是否有真的学过webpack，因为学过的话肯定会知道一两个loader和plugin的，虽然现在脚手架基本替我们搞定了。

答案:

# 常用loader：

  • file-loader（让webpack能处理文件）
  • url-loader（内置file-loader，并且能将文件转换为base64，缺点是只能处理图片和字体文件）
  • source-map-loader（加载额外的map文件，便于我们调试，但代码量会比不加载要多）
  • Image-loader（压缩图片文件）
  • babel-loader（最为核心的loader，将es6代码转换为es5）
  • ts-loader（将ts转换为js）
  • Awesome-typescript-loader（ts-loader的升级版）
  • sass-loader
  • css-loader（加载css，支持模块化/压缩/导入）
  • style-loader（将css代码注入到javascript中）
  • postcss-loader（自动加上兼容前缀，和autoprefixer插件配合使用）
  • eslint-loader（通过eslint检测js代码，有错会在页面进行警告）
  • tslint-loader（几乎同上）
  • vue-loader（加载vue文件）
  • less-loader
  • html-loader

# 不常用loader：

  • raw-loader（加载文件的原始内容，一般我们导入文件获取到的是对象，使用方式为!!raw-loader!文件路径）
  • Svg-inline-loader（将压缩后的svg内容注入代码）
  • Json-loader（能加载json文件，webpack默认包含）
  • mocha-loader（这个我只是听说但没真的用过，加载mocha测试用例代码的）
  • cache-loader（将loader进行缓存到磁盘）
  • I18n-loader（国际化）

# 常用plugin：

  • html-webpack-plugin（和html-loader相辅相成，简化html文件创建）
  • web-webpack-plugin（html-webpack-plugin升级版）
  • webpack-parallel-uglify-plugin（多进程执行代码压缩，提供构建速度）
  • mini-css-extract-plugin（分离样式文件）
  • speed-measure-webpack-plugin（可以看到每个plugin和loader的耗时）
  • webpack-bundle-analyzer（可视化webpack输出文件）
  • webpack- dashboard（展示相关打包信息）
  • wepback-merge提取公共配置，减少重复配置
  • size-plugin（监控资源体积变化）
  • hotmoduleReplacementplugin（模块热更新）

# 不常用plugin：

  • define-plugin（定义全局环境变量，但现在几乎不用，因为指定mode后可以自己对应的配置文件里面添加）
  • ignore-plugin（没用过，忽略部分文件用的）

# loader和plugin的区别

# loader

loader本质就是一个函数，对接收的内容进行处理，将内容转换成webpack能认识的js，也就是预处理工作

# plugin

基于事件流框架tapable，用来扩展webpack的功能，在运行时广播出许多事件，plugin监听到这些事件后就会在合适的时机通过webpack提供的api改变输出结果

# 总结

loader是解析规则，写在module.rules中，值为数组对象，对象中包含类型文件，loader，options等属性

plugin是功能扩展，写在plugins中，类型为数组，每一项都是plugin的实例，参数通过构造函数传入

# 模块打包原理

webpack为每个模块创造一个可以导出和导入的环境（将import改为require），本质上没有修改任何的代码逻辑，执行顺序和模块加载顺序没变

# 文件监听原理

webpack开启监听陌生的方式有两种，一种是webpack命令行加上--watch，一种是在配置文件加上watch：true

开启之后，webpack会在每次源码发生变化时构建出新的输出文件，但构建新文件这一步不会马上执行，而是在会将改变缓存起来，等aggreagetime设置的时间到后执行

有了新文件也不会刷新页面，需要我们手动刷新

# 热更新原理（HMR）

我们上面提到的手动刷新问题就是通过热更新机制来解决，这里用大白话的方式简单说下。

hmr的核心是客户端从服务端拉来更新后的文件，而实现这一步的是webpack-dev-server（简称为wds）与客服端建立起websocket，当本地资源发生变化的时候（文件系统检测）wds就会向浏览器推送更新，并带上构建的hash，客户端会对资源进行新旧比较。客户端发现差异后wds会发起ajax请求获取更改内容（是一个json），该josn包含了所有要更新的hash值，客户端就可以通过这些信息发起jsonp请求获取chunk的增量更新，也就是最新的模块代码，获取增量更新后就由hostmoduleplugin去决定如何处理如何保留如何更新。

# 配置文件的占位符

  • hash 随机hash值，只要项目文件有修改就会被
  • chunkhash 与webpack打包的chunk有关
  • contenthash 根据文件来生成的hash
  • ext 资源后缀
  • name 文件名
  • path 文件相对路径
  • folder 文件所在文件夹
  • Emoji 随机的指代文件的emoji

# 如何提高webpack构建速度

跟如何提升页面性能是同级别的问题，有必要好好记忆。

  1. 使用高版本的webpack和node
  2. 多进程构建（thread-loader）
  3. 压缩代码
    1. webpack-parallel-uglify-plugin（多进程压缩）
    2. uglifyjs-webpack-plugin 开启 parallel 参数 (被上面的淘汰了)
    3. terser-webpack-plugin 开启 parallel 参数
    4. css-loader开启minimize开启cssnano压缩css
  4. 图片压缩
    1. iamge-webpack-loader
    2. url-loader
  5. 缩小打包作用域
    1. exclude/include
    2. resolve.modules指明第三方模块绝对路径，减少不必要查找
    3. resolve.extensions减少后缀的尝试
    4. noparse忽略不需要解析的库
    5. 合理使用alias
    6. ignoreplugin忽略不需要的模块
  6. 提取公共资源
    1. html-webpack-externals-plugin将基础包通过cdn引入
    2. splitechunksplugin
  7. 利用缓存
  8. 清除没有引用过的模块
  9. 构建的代码有大量闭包，通过scope hostiing将所有痛苦按引用顺序放在一个函数作用域里面，并进行必要的重命名防止命名冲突
  10. 动态polyfill（就是动态的实现浏览器兼容，而不是一次性全给出）
