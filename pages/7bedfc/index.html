git日常工作命令

闲来无事，来整理下在平日工作中常用的命令。

在正式开始之前，我先来划分下等级，分别为频繁/偶尔

# 频繁

  • # git clone

    • 拷贝远程仓库

  • # git status

    • 查看仓库当前状态，显示文件状态，有四种状态：
      • Untracked 表示该文件是处于未跟踪状态，通俗点讲就是一整个文件都是新建的，还没加入了到 git 中，此时用 git add 可将其加入暂存区（临时库存），状态变更为 Staged
      • Staged 表示该文件已在暂存区 这时就可以 git commit 将文件丢入本地仓库，状态变更为 Unmodified
      • Unodified 表示该文件是为更改状态。同时该文件已在本地仓库，可直接提交到远程仓库。此外，此时的文件已经处于跟踪状态，一旦发生改变状态将会变成 Modified
      • Modified 对处于 Unmodified 状态的文件进行更改，文件状态将变更为 Modifed

  • # git add

    • 添加文件到本地仓库的暂存区。
    • 在本地修改文件之后，要把改变后的文件重新添加到仓库
    • git add 有三种用法：
      • git add . 添加当前目录所以文件到暂存区
      • git add [dir]。添加当前目录下指定目录到暂存区
      • git add file1 file2 ... 添加当前目录下指定的单个或多个文件到暂存区

  • # git commit

    • 将暂存区的文件添加到本地仓库
    • 常用的参数有-m，用来说明单次提交的备注信息git commit -m "feature-这次提交我提交xx新功能"
    • 同样的，git coomit 也能想 git add 一样指定文件添加

  • # git push

    • 将本地仓库上传到远程仓库并合并 （!注:请保证本地仓库提交前与远程仓库版本一致，或除所更改文件外全部相同，否则将会发生合并冲突无法推送成功）

# 偶尔

  • # git branch

    • 创建新分支 git branch 新分支名1
    • 创建的新分支是基于当前分支进行拷贝，也就是新分支拥有旧分支的所有代码

  • # git checkout

    • 切换分支 git checkout 新分支名1 (!注：在切换分支前，务必要先对原分支代码进行 commit，否则更改的文件会消失！虽然有抢救方案，但我也只是利用 git 的本地应用救回来的)

  • # git merge

    • 合并分支 git merge 新分支1(当前在master分支下，调用此命令就是将‘新分支1’的代码合并到master)
    • 合并代码时可能会出现一些冲突的情况，解决方案自行百度

  • # git pull

    • 合并远程代码 ，用来将合并代码与远程代码进行同步，也就是将本地仓库更新到远程仓库的版本
    • 两种常见用法:
      • git pull origin master:新分支1 将远程仓库master分支与本地仓库新分支1进行合并同步
      • git pull origin master 当合并的远程仓库分支和本地仓库分支相同时可以省去冒号

  • git config core.ignorecase

    • 区分大小写git config core.ignorecase false，开启大小写检验
