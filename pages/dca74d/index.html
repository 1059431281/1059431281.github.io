# configurable

其实这个文件并没有什么好介绍的，因为他不是 vueuse 提供出来的 api，只是在内部的 api 中使用到了而已。这里写出来的原因，是因为我之前有一些文章都写错了它。例如 defaultWindow，我把它的获取方式改成从"@vueuse/core"中获得，实际上并非如此。它是存在于 core 文件夹下一个可配置项的文件__configurable.ts，在源码中的正确的写法是../configurable

# 什么是 configurable？

vueuse/core 各文件夹会用到的，有关默认值的配置文件

# 源码分析

说实话这个么没什么好解析的，这种源码一看就知道是干什么的了，分别是导出变量类型和变量默认值 。

import { isClient } from "@vueuse/shared";

export interface ConfigurableWindow {
  /*
   * Specify a custom `window` instance, e.g. working with iframes or in testing environments.
   */
  window?: Window;
}

export interface ConfigurableDocument {
  /*
   * Specify a custom `document` instance, e.g. working with iframes or in testing environments.
   */
  document?: Document;
}

export interface ConfigurableNavigator {
  /*
   * Specify a custom `navigator` instance, e.g. working with iframes or in testing environments.
   */
  navigator?: Navigator;
}

export interface ConfigurableLocation {
  /*
   * Specify a custom `location` instance, e.g. working with iframes or in testing environments.
   */
  location?: Location;
}

export const defaultWindow = /* #__PURE__ */ isClient ? window : undefined;
export const defaultDocument = /* #__PURE__ */ isClient
  ? window.document
  : undefined;
export const defaultNavigator = /* #__PURE__ */ isClient
  ? window.navigator
  : undefined;
export const defaultLocation = /* #__PURE__ */ isClient
  ? window.location
  : undefined;

# 注意点

# isClient

关于这个 api 源码中藏的其实有点深，我这里直接列出它的源码，很精简

const isClient = typeof window !== "undefined"

意识是说当前实例下没有 window 的话就会返回 undefined，如果是 undifined 的话，后续在 configuration 中就不会直接取 window 而是 undfined，也就是没了默认值
