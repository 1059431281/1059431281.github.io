大白话diff算法

# 前言

diff算法是基于虚拟dom去实现的，而虚拟dom是一种js对象在，这里不做讲解，那么废话不多说马上开始diff极速版！

首先，diff算法的目的是什么？答案当然是进行最小化的更新，提高我们的页面性能。

题外话，浏览器自身也是有做大量dom修改时的优化，所以虚拟dom并不是为了diff算法服务的可别误解。

# diff算法的流程

  1. 数据改变触发setter，setter通过dep触发notify，notify触发了订阅者，订阅者开始执行patch比较
  2. 通过patch比较，发现两个元素一样则直接return，不一样则进入下一步
  3. 通过patchVnode进行比较，发现新旧内部一样则直接return，不一样则进入下一步
  4. 通过patchVnode发现不一样好有四种情况：
    1. 新旧vnode内部都是文本节点，直接用新文本替换旧文本
    2. 新vnode内部为空，旧不是，直接删除旧内容
    3. 新vnode内容不为空，旧不是，直接添加新内容
    4. 新旧vnode都有内容且都是节点，进入下一步
  5. 通过updateChildren对新旧vnode内部进行指针首尾比较，有五种情况：

    1. 新指针头和旧指针头相等

    2. 新指针头和旧指针尾相等

    3. 新指针尾和旧指针头相等

    4. 新指针尾和旧指针尾相等

    5. 以上都不想等，判断新指针头当前所对应的节点在旧中是否存在，存在即进行patchVnode，不存在了直接创建

    6. 不是情况，是补充说明，在命中上述某一个条件时会接着去执行patchVnode，以此去循环保证发现所有不同（指针会在每次比较之后超对应方向挪动）

# 总结

还是没总结，但这里补充一点，虚拟dom存在的真正意义时为了更好的跨度，进行js对象这种支持性很好。
