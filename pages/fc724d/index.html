文件分块上传和下载

# 前言

对以前没去学习的知识点进行补充。

# 使用场景

1、后台管理系统中，用户可能需要下载大型文件，如报表、日志文件、数据库备份等。通过将文件切片下载，可以提高下载速度和稳定性，同时允许用户中断下载并从中断处继续下载。上传同理

2、在图片上传场景中，用户可以选择多张图片进行上传。通过切片上传，可以加快图片上传速度，并实时显示上传进度。同时，在上传完成后，可以提供预览功能，让用户可以立即查看上传的图片。

3、视频上传和预览：对于较大的视频文件，切片上传可以确保上传过程可靠且高效。同时，可以实现上传进度的实时展示。上传完成后，通过切片下载技术，用户可以流畅地观看视频，无需等待整个文件下载完成。

4、云存储和云盘应用通常需要处理大量文件的上传。通过切片上传可以提高上传速度和稳定性，并允许用户中断并继续上传。

5、当用户需要下载云存储或云盘中的大型文件时，可以使用切片下载技术，加快下载速度并提供中断恢复功能。

6、通过将文件切片并进行预览，在线编辑，可以提供更好的用户体验。用户可以在不需完全下载文件的情况下，直接预览和编辑文件。

# 前置知识

在前端开发中，文件可以看作是数据流的一种形式，因此可以通过数据流的方式进行文件操作，这种方式又被称为文件流操作，包括读、写、删、重命名

因此，要分块上传下载，就是去组合和拆分文件流

# Blob

通俗的讲，用来存储大量二进制的整体数据块，可以把他当成一个文件对象，不可修改（file对象是Blob的子类）

# ArrayBuffer

原始的二进制对象（Uint8Array）

# FileReader

前端读取本地文件的工具类

# 分片上传

# 常规上传的缺点

1、大文件上传耗时长，可能出现请求超时

2、占用服务器和网络宽带资源，影响其他用户访问速度

3、上传中断，要重新上传

4、无法看见上传进度

# 实现思路

1、借由file对象是Blob的子类特性，因此可以使用到Blob.Prototype.slice来创造切片

2、借由切片创建formData对象传给后端。后端要临时存储切换，等传递完成再合并

        const handleChange = (e) => {
            const file = e.target.files[0]

            // 每个切片的大小
            const chunkSize = 200 * 1024 // 200kb （1mb = 1024kb = 1024 * 1024b）
            const totalChunks = Math.ceil(file.size / chunkSize);// 计算总切片数

            let start = 0;
            let end = Math.min(chunkSize, file.size);
            console.log('file',file);
            
            for (let index = 0; index < totalChunks; index++) {
        
                const chunk = file.slice(start, end); 

                // 要发送文件给后端的话，需要表单结构
                const formData = new FormData();
                formData.append('file', chunk);
                formData.append('totalChunks', totalChunks);
                formData.append("chunkIndex", index);
                formData.append("fileName", file.name);
                formData.append('isLastChunk', index === totalChunks - 1);

                // 把这个切片的文件发给后端
                fetch('xxxx/upload',{
                    method: 'POST',
                    body: formData
                })
                
                start = end;
                end = Math.min(start + chunkSize, file.size);
            }
            
        }
    </script>
</body>

# 断点续传，记录和恢复上传状态

# 实现思路

通过localStorage或sessionStorage来存储已上传信息，每次上传前，查询是否已经有存储对应的切片信息。

这里做个简易实现

        const handleChange = (e) => {
            const file = e.target.files[0]

            // 每个切片的大小
            const chunkSize = 200 * 1024 // 200kb （1mb = 1024kb = 1024 * 1024b）
            const totalChunks = Math.ceil(file.size / chunkSize);// 计算总切片数

            const storedUploadedChunks = localStorage.uploadedChunkIndex || 0

            
            
            for (let index = storedUploadedChunks; index < totalChunks; index++) {
        
                const chunk = file.slice(chunkSize * index,Math.min(chunkSize * (index + 1), file.size)); 
                const isLastChunk = index === totalChunks - 1;

                // 要发送文件给后端的话，需要表单结构
                const formData = new FormData();
                formData.append('file', chunk);
                formData.append('totalChunks', totalChunks);
                formData.append("chunkIndex", index);
                formData.append("fileName", file.name);
                formData.append('isLastChunk', isLastChunk);

                // 把这个切片的文件发给后端
                fetch('xxxx/upload',{
                    method: 'POST',
                    body: formData
                })
                
                if(!isLastChunk) localStorage.uploadedChunkIndex = index + 1;
                else localStorage.uploadedChunkIndex = null;
            }
            
        }

# 分片下载

# 常规下载的缺点

1、下载占用宽带，导致其他用户下载慢

2、下载出现错误，需要重新下载整个文件

3、大文件下载慢


        const downloadFile = () => {
            // 获取要下载文件信息
            fetch('/xxx/download',{
                method: 'GET',
                headers: {
                    'Content-Type': 'application/json'
                }
            })
            .then(res => res.json()) // fetch需要自己转
            .then(data => {
                const totalSize = data.totalSize // 总文件大小
                const totalChunks = data.totalChunks // 总切换数量
                const fileName = data.fileName
                const chunks = [] // 切片数组
                let downloadChunks = 0 // 下载的切换数

                // 开始下载切换
                for(let curChunkIndex = 0;curChunkIndex < totalChunks;curChunkIndex ){
                    fetch(`xxx/donwload/${curChunkIndex}`)
                    .then(res => res.blob())
                    .then(chunk => {
                        downloadChunks++;
                        chunks[curChunkIndex] = chunk // 存储切片

                        if(downloadChunks === totalChunks) {
                             // 合并切片
                             const  mergeBlob = new Blob(chunks)
                             
                            // 创建对象 URL，生成下载链接
                            const downloadUrl = window.URL.createObjectURL(mergedBlob);

                            const link = document.createElement('a')
                            link.href = downloadUrl;
                            link.setAttribute('download', fileName);
                            link.click();
                            // 释放资源
                            window.URL.revokeObjectURL(downloadUrl);
                        }
                    })
                }
            })
        }

# Range

我才知道有这么个东西，简单记录一些

Range 是 HTTP 协议中一个标准请求头，用来告诉服务器：

“我只要文件的某一部分字节，而不是整个文件。”

示例：

Range: bytes=0-999
1

代表：我要这个文件 第 0 ~ 999 字节（共 1000 字节）。

服务器若支持 Range，会返回：

206 Partial Content
1

并带上 Content-Range：

Content-Range: bytes 0-999/3000000
1
