# 前言

现今小程序的编写更偏向于用uni而非原生，因此在没有强制要求最好的兼容性前提下都会选择用更加简便的uni，因此本篇文章的代码参考也是基于uni。

# 如何使用uni开发小程序？

# 第一步

第一步自然是创建项目，而创建项目有两种方案，分别是用命令行和uni的一键创建。

命令行：vue create -p dclound/uni-preset-vue my-project

但我们这里不用命令行创建，而是用可视化一键（两者目录机构有些不同）

# 第二步

配置mainfest.json，设置小程序AppID

# 第三步

运行。

# 获取用户信息问题

首先，我们现在可以不可能再像以前那样，使用getUserProfile/getUserInfo去获取用户信息了，理由是微信因为一些实际场景原因，不想处理直接一刀切。现在再去用这个请求，只会返回一些没意义的东西。

# 微信支付

这里不会攥写服务端代码，但会多少提及一下思路流程。

在实际开发当中，我们调起支付的流程大致就是，想所需支付的商品信息发送给后台，后台会根据订单信息，去创建对应的订单，经过对单号和签名信息的处理，将需要的支付参数返回给小程序。

小程序在接受到必要的参数之后，通过wx.requestPayment发起微信支付。

wx.requestPayment({
  timeStamp: '', // 时间戳
  nonceStr: '', // 随机字符串，长度32位以下
  package: '', // 订单号标识，也就是订单号id —— 通过后端返回
  signType: 'MD5', // 签名算法，应与后台下单的一致  —— 通过后端返回
  paySign: '', // 签名 —— 通过后端返回
  success (res) { }, // 会在用户完成支付后调用
  fail (res) { } // 会在用户取消支付后调用
})

# 云开发

小程序官方很明确说了，如果我们是用云开发，那么我们完全可以省略掉上面的参数，只需要通过云函数进行统一下单，免证书免签名直接返回我们所需参数，只需要将返回值直接当作操作即可。

如果不用云开发，那小程序实际还会经过鉴权一系列步骤，在获得支付授权后才会调起支付界面，让我们去输入密码。

至于后续的步骤，还是好好看一遍上面的官方流程图，相当清晰。

！！注意！！

通常我们requestPayment的成功回调函数需要在我们点击完成后才会触发。有时候，我们希望后台的数据在成功之后就得到更新，而不是等到我们手动去点击触发更新。实际上，我们可以让后台帮我们个忙，当检测到支付成功的结果后，手动将结果的订单号交给服务器去做后续操作。

# 点金计划

全面提升了微信支付后的跳转能力，如果没开启的人需要去开启。

因为在做h5小程序面就遇到没开启的问题，他是针对微信支付后跳转的改造，在微信浏览器里面，如果没开启点金，那么我们支付后的跳转会直接闪退出去，小程序的话还不清楚会不会。

幸运的是，这项工作可以交给后端处理。

https://pay.weixin.qq.com/wiki/doc/apiv3_partner/open/pay/chapter3_5_1.shtml

# webview的使用

webview内部使用的webkit内核，但这里不是我们要说的重点。

我们知道，在小程序中，我们可以通过直接webview标签，直接使用我们h5网页，这无论是在成本还是简便性上都大大得到缩减。

虽然在api的使用上会有很多限制，但我们可以借此一次开发，既开发h5又开发了小程序，不过现在有uni不需要那么麻烦。

它在我们小程序中，最大的好处，我认为就是我们跟人或者跟其他项目有所关联的时候，只需要简单开发一下网页就行了。毕竟，开发成小程序的话，是没法嵌入，而且还要去发布也很麻烦。

# 前端

一般在前端使用正常会是这样

import wx from "weixin-js-sdk";
import wxjssdkConfig from "@/config/wxjssdk"
import { isWeixinBrowser } from "@utils/func"
import { fetch } from "@/http/axios" 

interface AnyObject {
  [ key: string]: any
} // 测试的时候不小心用ts构建项目，用anyObject方便跳过大多数检查。


class WxSdk implements AnyObject {
  constructor() {
    Object.assign((this as AnyObject).__proto__, wx) // 这一步，是为了我们将实例挂载到vue上后能直接调用wx里面的方法。
    this.getWxConfig();
  }
  getWxConfig() {
    if((!wxjssdkConfig.useWxSdkSign) || !isWeixinBrowser() ){
      console.warn('非微信浏览器，且非production或test，不能调用该接口: getWxConfig');
      return Promise.resolve('微信签名配置失败');
    }

    return fetch(wxjssdkConfig.signUrl, {currentUrl:window.location.href.split("#")[0]}, 'post') // 后端在进行签名的时候，需要我们项目所在地址，并且不带用#
    .then((res: any) => {
      wx.config({
        debug: false,
        ...res,
        jsApiList: wxjssdkConfig.jsApiList // 为数组，需要开启的接口功能，数组内没写的api是不能使用的
      })
      
      wx.error(err => {
        console.warn('微信配置失效', err)
      })
    })
  }
  share(){
    // ....
  }
}


export default {
  install: (app: any) => {
    app.config.globalProperties.$wxsdk = new WxSdk(); // vue3写法
    // vue.prototype.$wxsdk = new Wxsdk();  vue2写法
  }
}

# 后端

一开始在本地要想弄的时候，因为很少接触后端所以有点懵，最后在查阅资料后得出以下结论：

  1. 服务器需要在公(外)网上存在（也就是能通过网络访问的地址）
  2. 需要有一个公众号，最好是企业公众号，个人公众号认证问题比较麻烦

这两点是在本地调试的难点（作为像我这种菜鸟来说），因此有以下方法可以简化我们的难度：

  1. 利用NATAPP (opens new window)去实现将我们的内网穿透到外网。（虽然很多人会更推荐ngrok，但其实对于免费用户来说，免费版的网速慢到根本没法用）
    1. **官方教程路径：**https://natapp.cn/article/natapp_newbie
  2. 微信官方有提供我们测试号:https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login
    1. appID和appsecret可以在后端写死
    2. 接口配置信息，这个可以为空，只有在你需要服务端去接受微信服务端会返回一些消息信息时才需要填写。
    3. js接口的安全域名，这个最好是填写我们项目运行的本机ip，否则签名会出现no domain的错误

解决掉以上难点之后，推荐一个简单node签名库，基本不需要怎么改就可以拿出用作我们本地的服务器项目了：https://github.com/strongwray/wx-node-signature（有坑！看后面的定时器问题）

# 可能出现的问题

跨域

就算我们完成了以上的操作，还是可能会出现一些问题，首先是跨域，可以通过前端代理去修改，也可以改一下上面推荐库的代码，在wx_server.js中加入：

app.all("*",function(req,res,next){
  res.setHeader("Access-Control-Allow-Origin","*");
  res.setHeader("Access-Control-Allow-Headers","X-request-With,content-type");
  res.setHeader("Access-Control-Allow-Methods","GET,POST,DELETE,PUT,OPTIONS")
  next()
})

# 下载

在小程序中，要实现下载，那么就必须先获取对应的图片信息，然后再用改图片的本地路径去进行下载，不能直接通过网络路径实现下载。

uni.getImageInfo({
  src: "https://lovelijunyi.gitee.io/img/banner/1.jpg",
  success: (e) => {
    console.log(e)
    uni.saveImageToPhotosAlbum({
      filePath: e.path
    })
  },
  fail: (e) => {
    console.log("error:",e)
  }
})
