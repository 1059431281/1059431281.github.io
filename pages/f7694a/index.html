# useReducer

useReducer 是 useState 的替代方案，useState 能做到的事，它都能做到，甚至做得更好。useReducer 某种程度上解耦了操作逻辑(action)和后续的行为(一般是 UI 的更新)，虽然代码量变多了，但是看起来更加整洁。

  function reducer(state, action){
    switch(action.type){
      case "incr":
        return { count: state.count + 1 };
      case "redu":
        return { count: state.count - 1 };
      default:
        throw new Error();
    }
  }
  const [ state, dispatchNum ] = useReducer(reducer, initState)

# 生命周期小技巧

hooks的生命周期和原本的class，大致有这样的关系

Constructor ➡️ useState

render ➡️ 函数自身

getDeriverStateFromProps➡️好像没有

shouldComponentUpdate➡️好像没有

getSnapshotBeforeUpdate➡️好像没有

componentDidMount ➡️ useEffct-常规形式

componentDidUpdate ➡️ useEffect-给予第二个参数

componentWillUnmount ➡️ useEffect返回的函数

// routes/index.js
import { useRoutes } from "react-router-dom";
import { lazy } from "react"
// RouteObject
const Base = lazy(()=>import('@/page/base/index'))
const Other = lazy(()=>import('@/page/other/index'))
const Other2 = lazy(()=>import('@/page/other/other2.js'))

function GetRoutes() {
  const routes = useRoutes([
    {
      path: "/",
      element: <Base />
    },
    {
      path: "/other",
      element: <Other />,
      children:[
        {
          path: "/other/two",
          element: <Other2 />,          
        }
      ]
    },
  ])
  return routes;
}
export default GetRoutes

# 与vue的diif算法区别

两者的核心区别的话，vue的diff算法是局部创建新的虚拟dom对象，然后使用diff算法进行对比，进行局部更新。而react，是直接创建出一个全新的虚拟树，然后与上一次的dom对象进行对比找不同，最终再进行局部更新。

简而言之，一个生成局部的虚拟dom对象进行对比，一个是直接生成完整的虚拟树后进行对比。
