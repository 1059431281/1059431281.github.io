ajax基础

# Ajax

# 同步提交和异步提交的区别

# 前言

  • 表单具有默认的提交行为，默认是同步的，同步表单提交，浏览器锁死（转圈圈）等待服务端的相应结果
  • 表单的同步提交后，无论服务端响应的是什么，都直接把响应结果覆盖在当前页面

# 详细前言

  • ajax 是属于异步操作。之所以把这个知识点放在这里讲因为它跟 ajax 有很大关系

  • 在以前，没有 ajax 之前，页面上面的一些提示信息（比如说密码错误等等）甚至都是直接渲染在页面上。

    • 注意！这个渲染是直接覆盖了原本的页面，也就是相当于刷新了页面
    • 此外，为什么我们上面说甚至，原因是更多情况都是用户提交的数据渲染在了页面上（其实没差）

  • 总之，同步提交会造成原页面被覆盖的弊端，因而才要使用异步提交

  • 异步提交页面不会刷新，交换方式非常灵活

  • 以前没有 ajax 都是这么干的，甚至有些直接就是渲染了提示信息出来了

  • 异步提交页面不会刷新，交互方式更灵活

# 中言

  • 除了 ajax 之外，当然还有其他解决方法
  • 比如在表单提交后，页面自动重新加载当前页面即可
    • 当然了，这种重新加载属于 post 请求，因为我们要以你提交的数据为前提进行重新加载

# XMLHttpRequest

​ XMLHttpRequset 这个对象的主要功能便是实现与服务器交互，让你可以从 url 中获取数据，而不需要重刷新页面。他的好处，可以在不影响用户使用的时候更新新的页面内容(数据)。ajax 编程中，他是非常火爆的存在。

# XMLHTTPRequest.open()

  • open()方法的作用是初始化一个请求
  • 它有五个参数(method，url,async,user,password)
    1. 设置要使用的请求方法，一般有 get 和 post——必选
    2. 要发送请求的 url——必选
    3. 布尔值，true 表示要使用异步操作、false 则是同步操作——可选
    4. 用于设置用户的账号认证——可选，默认为 null
    5. 用于设置用户的密码认证——可选，默认为 null
