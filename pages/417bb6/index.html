webpack常用插件使用

基本上，例子中的代码，有关webpack都是基于vue-cli封装后的。

# webpack-bundle-analyzer

可视化webpack构建后的输出文件，有利于我们观察项目的详细体积。

// vue.config.js

const webpackBundleAnalyzer = require("webpack-bundle-analyzer").BundleAnalyzerPlugin;

modules.exports = {
  configureWebpack:{
    plugins:[
      new webpackBundleAnalyzer()
    ]
  }
}

# thread-loader

开启多进程构建，简单说就是开启多个worker池去运行不同的loader。

记得，只有耗时长的loader才去开启多进程，否则维护worker池也是有消耗的。

const path = require("path");

modules.exports = {
  configureWebpack:{
    module:{
      rules: [
        {
          test: /\.js$/,
          include: path.resolve("src"),
          use:[
            "thread-loader",
            "babel-loader"
            // 将需要的进入多进程的放在thread-loader后面
          ]
        }
      ]
    }
  }
}

# parallel

vue-cli有进行封装

  • Type: boolean

  • Default: require('os').cpus().length > 1

    是否为 Babel 或 TypeScript 使用 thread-loader。该选项在系统的 CPU 有多于一个内核时自动启用，仅作用于生产构建。

module.exports = {
  parallel: true
}

# webpack-parallel-uglify-plugin

要注意一个问题，这个压缩插件貌似存在着一个问题，需要对webpack进行降级处理，降到4.0.0这样，否则会出现找到tapsync的情况。

多进程压缩代码，找个网络代码示例：

const ParallelUglifyPlugin = require('webpack-parallel-uglify-plugin');

new ParallelUglifyPlugin({
      uglifyJS: {
        output: {
          comments: false,//是否保留代码中的注释，默认为保留
        },
        warnings: true,//是否在UglifyJS删除没有用到的代码时输出警告信息，默认为false
        compress:{
          drop_console: true,//是否删除代码中所有的console语句，默认为false
          collapse_vars: true,//是否内嵌虽然已经定义了，但是只用到一次的变量， 默认值false
          reduce_vars: true,//是否提取出现了多次但是没有定义成变量去引用的静态值，默认为false
        }
      },
      cacheDir: '',//用作缓存的可选绝对路径。如果未提供，则不使用缓存。
      sourceMap: config.build.productionSourceMap,//可选布尔值。是否为压缩后的代码生成对应的Source Map(浏览器可以在调试代码时定位到源码位置了),这会减慢编译速度。默认为false
 }),

# 题外话：缩小打包范围

// vue.config.js

// 锁定js的打包范围
modules.epxorts = {
  // 防止 webpack 解析那些任何与给定正则表达式相匹配的文件。
  noParse: noParse: (content) => /jquery|lodash/.test(content),
  congiureWebpack: {
    plugins: [     
      // 忽略不需要的模块，让它不要进行打包
      new webpack.IgnorePlugin({
          resourceRegexp: /^\.\/locale$/, // 匹配资源文件
          contextRegExp: /moment$/    // 匹配检索目录
      })
    ]
    module: {
      rules: {
        test: /\.js$/,
        include:"",
        exclude:"",
      }
    },
    resolve: {
      extensions: [],
      alias: {}
    },
    noParse: {
      
    }
  }
}

# html-webpack-externals-plugin

当我们需要引用一个库，但又不希望被webpack打包，还不影响我们的cmd/amd/全局使用，我们可以借助该插件将基础包引入，最好是通过cdn。

它最后的结果，就是动态在我们index.html中加入script标签。

const HtmlWebpackExternalsPlugin = require('html-webpack-externals-plugin');

module.exports = {
  configureWebpacl: {
    plugins: [
            new HtmlWebpackExternalsPlugin({
        externals: [{
          module: 'vue',
          entry: 'https://unpkg.com/vue@3/dist/vue.global.js',
          global: 'Vue'
        }]
      })
    ]
  }
}
