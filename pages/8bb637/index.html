ts学习总结--基础篇

对于ts的学习总结，首先是基础篇

# 什么是ts？

ts又称为typescript，是一个建立在js之上的强类型语言，支持js所支持的es6语法，支持面向对象的概念，如类、接口、继承、范型等等。ts是不能直接在浏览器上运行，需要先将其编译为js才可。通常，我们可以用ts-node直接在本地运行。

# 为什么要使用ts？

在传统的js开发中，随着项目越发复杂庞大，我们对项目的认知和约束会越发模糊，虽然可能可以用相应的文档解决这个问题，但文档能提供的作用是有限的，而且维护一个那么庞大的项目文档并不容易，并且就真的去写文档，不可能把每个地方的每个细节都去撰写成文档。

为了解决上面所说的问题才出现了ts，ts增加类型约束的功能，可以在开发人员编写代码的时候进行类型检测，第一时间给出反馈，使得我们代码的质量/可读性/健壮性更好。

有了ts，他的类型检测就相当于起到了文档的作用，无论是自己还是他人接手项目都容易上手了不少。除此之外，ide的自动填充和自动联想都起到了很好作用。

# ts中any、never、unknown、null、undefined之间的区别是什么

any：动态的变量类型，简单说就是失去类型检查作用。

let a:any；  // 等于任何值都行
1

never：直译的意思为从不/永不，也就是永不存在的类型。

说说它出现的意义，为了避免新增的联合类型没有对应的实现，写出类型绝对安全的代码

它是所有类型的子类型，因此never可以赋值给所有类型，反之任何类型不能复制给never

在有明确never类型返回类型的函数中，只要有return语句就必须有never类型的表达式，并且终点必须是不可执行的

never类型是针对抛出异常或永不有返回值的函数表达式的返回值类型

类型绝对安全的代码

interface Foo {
  type: 'foo'
}
interface Bar {
  type: 'bar'
}
interface Baz {
  type: 'baz'
}
type All = Foo | Bar | Baz
function handleValue(val: All) {
  switch (val.type) {
    case 'foo':
      // 这里 val 被收窄为 Foo
      break
    case 'bar':
      // val 在这里是 Bar
      break
    default:
      // val 在这里是 never
      const exhaustiveCheck: never = val 
      // 不能将类型“Baz”分配给类型“never”
      // 借由任何类型的值都没法传递给never，我们就能知道漏写了
      break
  }
}

# ts的协变、逆变、双变、抗变

协变和逆变，都是用来表示父类子类在安全类型转换后的兼容性。

但是，并非有extends才能实现这一点，只要结构相同就可以，因此下面的例子以结构相同为主而不是extends

# 协变

ts对象的兼容性是协变，也就是Y 类型可以赋值给 X 类型的情况就叫做协变，这便是X兼容Y（简单说，就是被赋值的一方被包含于赋值的一方）

interface X {
  name: string
}
interface Y {
  name: string,
  age: number
}
let x:X = {
  name: "小明",
}
let y:Y = {
  name: "小红",
  age: 18
}
x = y
y = x // 这样不行
// 此外，x = {name:"小红",age:18}直接这样也是不行的

# 逆变

printY = printX，函数x类型可以赋值给函数y类型，因为函数Y在调用的时候参数是按照Y类型约束，但是采用的函数内容是X里面的。这种特性便叫逆变，函数的参数有逆变的性质。（简单说，就是被赋值的一方包含于赋值的一方）

let printY: (y: Y) => void
printY = (y) => { console.log(y.age) }
let printX: (x: X) => void
printX = (x) => { console.log(x.name) }
printY = printX
printY(y) // 输出的是 y里面的name，也就是小红

# 双变

X=Y,Y=X，就是父类型可以等于子类型，子类型可以等于父类型，也就是即逆变又协遍（只有当关闭严格类型检查，才会有双变），这是不安全，所以一般不要

# 抗变

非父子类型之间不会发生型变，只要类型不一样就会报错（但结构相同的，其实还是会）

# Exclude

简单解释，就是判断出第一个参数是否为第二个参数的子类，是则返回never，否则返回第一个参数（下面的例子没有用extends方式，因为只要结构一样也是可以的）

interface x{
  a: string;
  b: number;
  c: boolean;
}
interface y{
  a: string;
  b: number;
}
type newType = Exclude<y,x>// 等同于 type newType = y
type newType = Exclude<x,y>// 等同于 type newType = never

type A = number | string | boolean
type B = number | boolean
type newType = Exclude<A,B>

# 枚举和object的区别

  1. 取值上的区别。枚举可以用键取值，值取键，对象只能用键取值。
  2. 默认值的区别。枚举在不指定初始值的情况，会从0递增，对象不会。
  3. keyof上的区别。在运行时枚举是真实存在对象，但想要获取它的属性名不能单纯用keyof，需要keyof typeof
