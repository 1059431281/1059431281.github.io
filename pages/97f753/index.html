sass-loader的一些情况

因为无论是在实际工作中还是平时新构建项目，都会遇到按照sass出现问题，而其中发生问题的原因关键就在node/sass-loader/node-sass之间的版本关系。

# 核心问题

其实关于这些，核心的问题就是node-sass这个包的问题，这个包并没有对node实现向后兼容什么的，所以才会导致一系列的问题出现。

# 如果安装不了node-sass

npm/yarn config set registry https://registry.npm.taobao.org

改一下源。

# node-sass对应的版本

这玩意，其实node-sass的npm上面就有

# sass-loder

这个的话，要注意的一个点就是，sass-loader13开始，应该往webpack@5开始用了。

# node版本管理器

mac用n/nvm

window用nvm

# 最好的解决方案

其实，比起在node-sass上纠结，还不如直接用sass，它原本的名称是dart-sass，后续改名为sass。

它对node对兼容性很好，所以通常不会出现问题。

但是，通过网络冲浪发现可能有一些潜在问题：

  1. 可能会对element-ui图标乱码
  2. 一些第三方组件库会抛警告
  3. 可能会导致/deep/无法使用，只能使用:deep()

尽管如此，这些都只是可能问题，2022之后的项目，都可以考虑直接使用sass和sass-loader搭配
