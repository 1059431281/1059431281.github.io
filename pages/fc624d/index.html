长列表性能优化

# 前言

其实对于长列表的性能处理，在像是订单列表，评论区、优惠券列表等场景是必不可少的。核心问题就两点：dom元素过多，导致页面卡顿；数据过多，接口返回时间慢，页面数据不好处理。接下来，就针对这两种情况进行处理。

# 优化方案

目前主流的方案就是“分页加载”+“虚拟列表“

# 虚拟列表

分页加载这种是比较常见的，所以这里说明一下虚拟列表。

简单来说，虚拟列表就是把只渲染“缓冲区”和“可视区”的元素，我们要做的就是区动态计算要渲染的列表区间，达到节省元素的目的。

由于这里是简易实现，所以采用列表项固定的场景。

<script setup>
import { ref,computed, reactive } from "vue"
// 页面容器高度
const SCROLL_VIEW_HEIGHT = 500;
// 列表项高度
const ITEM_HEIGHT = 50;
// 预加载数量
const PRE_LOAD_COUNT = SCROLL_VIEW_HEIGHT / ITEM_HEIGHT;

const containerRef = ref(null)

/** 模拟大量数据源 */
const sourceData = ref([])
const createSourceData = () => {
  sourceData.value = Array.from(Array(4000).keys())
}
createSourceData()

// 计算列表项产生后的列表容器总高度
const scrollViewHeight = computed(() => {
  return sourceData.value.length * 150
})

// 列表项显示的数据区间（头部预加载 + 可视窗口 + 尾部预加载）
const showRange = reactive({
  start:0,
  end:10
})

// 可视窗口的列表项数组
const currentViewList  = computed(() => {
  return sourceData.value.slice(showRange.start, showRange.end).map((el, index) => ({
    data: el,
    index,
  }));
})

// 监听页面容器滚动，计算数据区间
const onContainerScroll = (event) => {
  event.preventDefault();
  calculateRange();
};

const calculateRange = () => {
  const element = containerRef.value;
  if (element) {
    const offset = Math.floor(element.scrollTop / ITEM_HEIGHT) + 1;  // 计算距离顶部的距离可容纳的列表项数量
    const viewItemSize = Math.ceil(element.clientHeight / ITEM_HEIGHT); // 激素按容器内可容纳的列表项数量
    const startSize = offset - PRE_LOAD_COUNT; // 计算实际要预加载的数据开始位置
    const endSize = viewItemSize + offset + PRE_LOAD_COUNT; // 计算实际要预加载的数据结束位置
    showRange.start= startSize < 0 ? 0 : startSize
      showRange.end= endSize > sourceData.length ? sourceData.length : endSize
  }
}

// 滚动偏移量
const scrollViewOffset = computed(() => {
  return showRange.start * ITEM_HEIGHT;
});


</script>

<template>
  <div ref="containerRef" :style="{
    height: SCROLL_VIEW_HEIGHT + 'px',
    overflow: 'scroll',
  }"
    @scroll="onContainerScroll"
  >
    <div :style="{
      width: '100%',
      height: scrollViewHeight - scrollViewOffset + 'px', // 减去滚动偏移量部分，否则会出现底部空白
      marginTop: scrollViewOffset +'px',
    }">
      <div v-for="value in currentViewList" className="showElement" :key='value.data'>
        Current Position: {{value.data}}
      </div>
    </div>
  </div>;
</template>

<style scoped>
.showElement {
    display: flex;
    justify-content: center;
    align-items: center;
    border: 1px solid #000;
    margin-bottom: 8px;
    border-radius: 4px;
}

</style>

# 虚拟列表+分页

配上分页其实也很简单，就是判断一下是否触底，触底后增加数据，然后再去触发计算区间函数就行。

当然，这样做的缺点也很多，比如：

滚动事件可以添加节流事件避免造成性能浪费。

列表项高度不固定需要给定一个默认高度后设置新的高度在重新刷新容易截取的开始和结束位置。

滑动过快出现白屏问题可以尝试动态加载loading显示过渡，优化一些细节体验。

列表项中存在阴影元素需要考虑缓存处理，不然滚动时必然会引起重新加载。

# 方案劣势

虚拟列表在列表内容不多的情况下属于负优化

虚拟列表会触发重排，也会产生性能问题

# 最后

实际上，市面上已经有很成熟的开源库能解决这种问题，比如vue的vue-virtual-scroller，所以我们其实只要大致理解原理，不需要从零去开始。
