git中https克隆项目和ssh-key密码错误问题

这是我在 b 站评论区看见的一个评论，真的解决了我以前用 https 克隆仓库的一大痛点

从 2021 年 8 月 13 号开始 github 旧不再支持使用密码验证的方式去提交代码，只能使用 personal access token，也就是 ssh 免密秘钥.

至于变成生成秘钥，我找了篇博客https://blog.csdn.net/qq_38163309/article/details/105335097，当然可以直接去b站看“【狂神说Java】Git最新教程通俗易懂”中的p11，有详细讲到。

那么现在问题来了，既然现在彻底改用 ssh token 去登录，克隆项目页改用第二种 ssh 方案，那么之前用 https 方案 clone 的项目要怎么办呢？重新用 ssh 方式去 clone 然后该代码？那样太蠢了，我们只需要进入隐藏文件夹.git 中的 config 文件将 url 选项由url = https://.... 改成 git@....即可
