vite一点学习知识

# vite自带功能

esbuild运行时快速构建、rollup静态分析能力、cssnano的css提取和压缩优化、 build.terserOptions.compress.drop_console = true 去掉 console.log。小资源自动转化为base64、大资源用hash命名、默认做vendor拆分（把第三方库进行拆分、应用拆分，就是路由页面那些，还有最小拆分策略）

# 文件压缩

vite本身不提供该功能，主要是因为在生产环境下，压缩通常是由服务器端来处理的，而不是由打包工具来处理。这避免了重复的压缩工作，也让部署更加灵活。

如果非要不可，可用vite-plugin-compress

# vite-plugin-cdn-import

作用是让你在构建时将一些依赖（如 Vue、Element Plus、Axios 等）通过 CDN 引入，从而：

  • 减小打包体积
  • 提升构建速度（不再打包这些库）
  • 更快的首次页面加载（CDN 通常缓存和传输更快）

# esbuild和rollup

rollup的tree shaking、代码分割、生态、压缩更为优秀

# 为什么生产环境仍需打包（来源vite官网）

尽管原生 ESM 现在得到了广泛支持，但由于嵌套导入会导致额外的网络往返，在生产环境中发布未打包的 ESM 仍然效率低下（即使使用 HTTP/2）。为了在生产环境中获得最佳的加载性能，最好还是将代码进行 tree-shaking、懒加载和 chunk 分割（以获得更好的缓存）。

要确保开发服务器和生产环境构建之间的最优输出和行为一致并不容易。所以 Vite 附带了一套 构建优化 (opens new window) 的 构建命令 (opens new window)，开箱即用。
