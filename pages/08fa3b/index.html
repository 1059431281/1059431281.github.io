vue3虚拟dom和diff算法

# 虚拟dom

vue3当中，无论是虚拟dom还是diff算法都被重写了，但要实现的事情还是基本一样，但为了实现对应的功能之间细节发生了变化。

# 静态标记

在vue2当中，我们一旦生成新的虚拟dom，那就一定会全部都进行diff算法对比，但在vue3当中，有静态标记的虚拟dom是不会去进行diff算法对比。

总结来说就是：vue2——全量diff，vue3——非全量diff

# 如何做静态标记

vue3在生成虚拟dom的时候，会根据是否有对应的引用或绑定去给予该虚拟dom类型，具体来说就是在创建虚拟dom时的第四个参数，比如：

<div>1</div>
<div>
  {{ num }}
</div>

4

对应转换成的大概是：

...
  _createVNode("div",null,"1")
  _createVNode("div",null,_toDisplayString(_ctx.num),1)
...
1

3
4

第四个参数，为空时默认为0，表示是静态的，至于第四个参数的取值有很多，1-10分别对应不同的标记，像这里1表示的就是文本。

如果不只是文本，还有动态属性的话，比如说动态类，那就是数字2，接着会跟文本1进行与运算。

# 静态提升

上面提到了静态标记，既然是静态的东西，不会变，那为什么不将它提出去，以便于我们后续复用呢？

# 事件缓存

跟静态提升的目的其实一样，实现复用。

但有点不同，原本事件应该是动态，我们这里其实就是把他弄成了静态的。

# 如何实现？`

简单，用一个数组去维护一下就行了，把事件存在数组里面，后续直接拿出用就好。

<button @click="handleClick">
  点击

# diff算法

vue2中用updateChildren是新旧vnode都有内容且都是节点还不想等之后的比较，而在vue3中则是改用patchKeyedChildren了。

patchKeyedChildren对updateChildren进行了缩减，不需要头尾比较什么的，而是：

新头和旧头、新尾和旧尾，还有就是基于最长递增子序列进行移动/添加/删除

具体来说：

用沐华大大的例子吧

  • 老的 children：[ a, b, c, d, e, f, g ]
  • 新的 children：[ a, b, f, c, d, e, h, g ]
  1. 先进行头和头比，发现不同就结束循环，得到 [ a, b ]
  2. 再进行尾和尾比，发现不同就结束循环，得到 [ g ]
  3. 再保存没有比较过的节点 [ f, c, d, e, h ]，并通过 newIndexToOldIndexMap 拿到在数组里对应的下标，生成数组 [ 5, 2, 3, 4, -1 ]，-1 是老数组里没有的就说明是新增
  4. 然后再拿取出数组里的最长递增子序列，也就是 [ 2, 3, 4 ] 对应的节点 [ c, d, e ]
  5. 然后只需要把其他剩余的节点，基于 [ c, d, e ] 的位置进行移动/新增/删除就可以了
