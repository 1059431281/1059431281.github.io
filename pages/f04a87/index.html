# 简单说下vuex的实现

# 首先，什么是vuex？

虽然是题外话，但这里还是说一下。

所谓的vuex简单说就是一个大型的，应用于vue的状态管理器，通常在项目中的应用是多组件间的共享数据，以及全局信息的使用。

通常来说，在项目中复杂传递较低的时候，我们会使用他的下位替代bus。

  Vue.mixin({
    // 在数据初始化之前进行挂挂载
    beforeCreated(){

        this.$store = typef opionts.store == "function" ? options.store() :	options.store
      }else if(options.parent && options.parent.$store){
        this.$store = options.parent.$store;
      }
    }
  })
}

# 实现vuex的第三步

实现完install后，就要来实现核心的vue.store了。

因为该方法比较复杂，所以这里就直接把代码列出来(当然不看代码也没关系)，然后简单的讲解


class Store{
  constructor(options={}){
    this.options = options
    this.getters = {};
    this.mutations = {};
    this.actions = {};
    this._modules = new ModuleCollection(options)
    this.commit = (type,param)=>{
      this.mutations[type].forEach(fn=>fn(param))
    }
    this.dispatch = (type,param)=>{
      this.actions[type].forEach(fn=>fn(param))
    }
    const state = options.state;
    const path = [];
    instalModule(this,state,path,this._modules.root)
    if(Vue.observable){
      this.vmData = {
        state: Vue.observable(options.state || {})
      }
    }else {
      this.vmData = new Vue({
        data: {
            state: options.state
        }
      });
    }
    console.log(this)
  }
  get state(){
    return this.vmData.state ? this.vmData.state :this.vmData._data.state 
  }

}

// 提取通用的foreach为公共函数
function forEachValue(obj,fn){
  Object.keys(obj).forEach(key=>fn(obj[key],key))
}
// 注册getter
function registerGetter(store,getterName,getterFn,currentModule){
  Object.defineProperty(store.getters,getterName,{
    get:()=>{
      return getterFn.call(store,currentModule.state)
    }
  })
}
// 注册mutations
function registerMutation(store,mutationName,mutationFn,currentModule,){
    let mutationArr = store.mutations[mutationName] || (store.mutations[mutationName] = []);
    mutationArr.push((payload)=>{
      mutationFn.call(store,currentModule.state,payload)
    })
}

// 注册actions
function registerAction(store,actionName,actionFn){
    let actionArr = store.actions[actionName] || (store.actions[actionName] = []);

    actionArr.push((payload)=>{
      let res = actionFn.call(store,store,payload)
      if (!(res && typeof res.then === 'function')) {
        res = Promise.resolve(res);
      }
      return res

    })
}



// 模块收集
class ModuleCollection{
  constructor(rootModule){
    this.register([],rootModule)
  }
  register(path,rootModule){
    const newModule = {
      _rootModule: rootModule,
      _children:{},
      state:rootModule.state
    }
    if(path.length === 0){
      this.root = newModule;
    }else{
      const parent = path.slice(0,-1).reduce((module,key)=>{
        return module._children(key)
      },this.root)
      parent._children[path[path.length - 1]] = newModule
    }

    if(rootModule.modules){
      forEachValue(rootModule.modules,(rootChildModule,key)=>{
        this.register(path.concat(key),rootChildModule)
      })
    }
  }
}

ModuleCollection.prototype.getNamespace = function getNamespace (path) {
  var module = this.root;
  return path.reduce((namespace,key)=>{
    module = module._children[key]
    return namespace + (module._rootModule.namespaced ? key + '/' : '');
  },'')
};

// 递归状态树，挂载getters，actions，mutations
function instalModule(store,rootState,path,rootModule){
  const namespace = store._modules.getNamespace(path);
  if (path.length > 0) {
    const parent = path.slice(0,-1).reduce((state,key)=>{
      return state[key]
    },rootState)
    Vue.set(parent, path[path.length - 1], rootModule.state)
  }

  // 循环注册包含模块内的所有getters
  let getters = rootModule._rootModule.getters
  if (getters) {
    forEachValue(getters, (getterFn, getterName) => {
      getterName = namespace + getterName;
      registerGetter(store, getterName, getterFn,rootModule);
    });
  }
  let mutations = rootModule._rootModule.mutations
  if (mutations) {
    forEachValue(mutations, (mutationFn, mutationName) => {
      mutationName = namespace + mutationName;
      registerMutation(store, mutationName, mutationFn,rootModule)
    });
  }
  let actions = rootModule._rootModule.actions
  if (actions) {
    forEachValue(actions, (actionFn, actionName) => {
      actionName = namespace + actionName;
      registerAction(store, actionName, actionFn,rootModule);
    });
  }
  forEachValue(rootModule._children, (child, key) => {
      instalModule(store, rootState, path.concat(key), child)
  })

}

# 单一vuex的实现

假如是单一vuex，也不就是不包括模块的实现步骤是这样的：

  1. 在类中初始化getters/mutations/actions对象，以及获取实例化时传递过来的options参数对象
  2. 通通过options参数对象，去注册getters/mutations/actions（为了方便区分，接下来把参数对象中的getters等叫做参数getters等）
  3. 实现getters注册。首先遍历参数getters，随后通过Object.defineproperty(getters对象，参数getters遍历出来的键名)，随后给他绑定一个get方法方法去调用参数getters的方法，传指定参数即可
  4. 实现mutations/actions注册，与上述差不多，通过遍历参数对象往actions/mutations对象绑定方法，并传递合适的参数
  5. 实现commit/dispatch，很简单的去调用muations/actions对象内的方法就好
  6. 利用Vue.observable或new Date实现state响应式
  7. 给store提供一个get state方法

生成状的状态书格式会是

const newModule = {
    // 根模块store
    _rootModule:store,
    // 子模块
    _children:{
        moduleA:{
          _rootModule:moduleA,
          _children:{},
          state:moduleA.state
        },
        moduleB:{
          _rootModule:moduleB,
          _children:{},
          state:moduleB.state
        }
    },
    // 根模块状态
    state:store.state

生成这样格式之后的状态树我们主要将他进行安装就好，具体安装这里就不再多写，简单总结一下就是：

  1. getters/mutations/actions这些因为存在着重名情况，所以使用数组去绑定方法
  2. commit/dispatch的使用，因为方法变成数组，我们去使用时要改成遍历数组后去使用

# 命名空间

然后，为了避免重名问题便引入了命名空间，他实现的关键在于给绑定方法时加上模块命的前缀，但这里面有个难点，那就是内部vuex中使用不加前缀是被允许的，而使用有前缀的反而是不被允许的。

解决这难点的思路，大概就是额外再去创一个专属于命名空间的对象方法去进行维护，这里就不做拓展了，因为自己也没特别明白。

# 总结一下面试被问的说法

# 什么是vuex？

答：应用于vue的中央状态管理器，是将数据存在对橡树的变量当中，vue应用时直接读取这个变量即可，对外提供关键的install/store方法，并为了方便操作提供了快捷的map等方法，利用状态树转换实现模块化应用。此外，他是一次性，也就是会随也页面的关闭而消失

# vuex是怎么实现长缓存的？

答：日常使用是利用vuex-persistedstate插件，而插件原理是基于cookie一类

# mutation和actions的区别

答：直接一点的话，关键的区别就是muations中不能使用异步，而actions里是可以的。当然，这里并不是绝对，因为之所以不在mutations中使用异步，主要是为了我们方便在devtool中查看日志。

# 那么问题来了，为什么actions的异步就可以被记录？

答：因为action修改数据也需要通过提交mutation，而mutation中的方法又是同步的。所以devtools可以记录的到！重点是时机！
