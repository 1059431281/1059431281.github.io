# async与await详解

本文主要是根据模拟async和await使用时源码所进行的流程进行详解

首先，这里给出一个async和await的基本使用

      function a(){
        console.log("输出a")
      }
      function b(){
        return new Promise((resolve)=>{
          resolve('输出b')
        })
      }
      function c(){
        return new Promise((resolve)=>{
          resolve()
        }).then(v=>{
          return "输出c"
        })
      }

      function d(){
        return new Promise((resolve)=>{
          resolve()
        }).then(()=>{
          console.log("最后输出d")
        })
      }
      async function async1(){
        let valb = await b()
        let valc = await c()
        console.log(valb)
        console.log(valc)
        d();
        a();
      }

# 需要提前明白的知识

首先，不管你知不知道，async是由生成器（generator）和自执行函数组成。

然后，经过babel编码后的await和yield其实是一个东西，有兴趣可以去试着编译看看

最后，传统的生成器是需要我们手动去next，所以自执行函数就是把我自动去执行next，下面是一段自执行函数的代码参考

      function spawn(genF) {
        // async函数最后肯定会返回一个promise，所以这里返回一个promise，同时也是为了方便我们去链式操作
        return new Promise(function (resolve, reject) {
          // 这里的genF是我们传进来的生成器，这里将它进行实例化
          const gen = genF();
          // 这里是代码的核心函数，不断调用此函数next下去
          function step(nextF) {
            /* 
              这里的nextF可以用很多种传递方式，这里采用的是函数形式，会在匿名参数中调用next以进行下一步并将
              结果返回
            */
            let next;
            try {
              // 这里调用
              next = nextF();
            } catch (e) {
              return reject(e);
            }
            /*
             判断生成器是不是到结尾了，是的话就直接调用resolve结束promise，但是其实done为true的情况其实
             value的值也就是undifined了，除非你的生成器最后是以return结尾
            */
            if (next.done) {
              return resolve(next.value);
            }
            /* 
              这里便是整个自执行函数最核心的地方了，判断不是done之后将每次的next.value结果作为promise去构成
              then链式，实现的核心在于使用promise.resolve将值转为promise，当然了，这里其实也可以判断value
              是否是promise
            */
            Promise.resolve(next.value).then(
              function (v) {
                step(function () {
                  /* 
                    至于这个地方，便是我们await获取值的关键了。这里本人也踩了些坑，因为当时并不是清楚生成
                    器，所以苦恼了许久await是怎么拿到值的。实际上，在生成器当中，我们调用next后会在一次
                    yield停下，接着再调用一次会在下一个yield停下，而此时如果我们在调用next的时候给予参数，
                    后面上一个yield就会获取这个参数作为返回值 
                  */
                  return gen.next(v);
                });
              },
              function (e) {
                step(function () {
                  return gen.throw(e);
                });
              }
            );
          }
          // 初始化调用，值传undifined
          step(function () {
            return gen.next(undefined);
          });
        });
      }

# 进入正题

现在，我们将刚刚的代码基于自执行函数进行转换，变成以下样子

        function async1(){
          return spawn(function* (){
            let valb = yield b()
            let valc = yield c()
            console.log(valb)
            console.log(valc)
            d();
            a();
          })
        }

最后经过代码的不断拼接，总之就是去走spawn这个函数，流程大概就是

  1. 第一次next，执行到第一个yield停下，执行b函数
  2. 第二次next，在第一次next中的b函数所resolve的值会在此时返回给valb，剩下的与第一次同理
  3. 第三次next，在第二次next中的b函数所resolve的值会在此时返回给valb，然后因为done是true，会一直执行到结束，至于d中的then，因为没有用next去链式起来，所以身为微任务的他会比a更晚执行

# 总结

async/await的核心实际上只要理解了生成器和自执行函数就能轻易搞定，下面是几个平时需要注意的点

  1. await后面必须跟promise这句话并不全对，是因为标准的使用应该那样，否则就失去了使用它的意义了
  2. async会返回promise，所以我们可以利用这个性质在一个声明一个函数a，a是async，并使用await了，然后再再定义abox函数，在里面使用await 函数，省去我们手动return promise的功夫
  3. 想要了解async/await这些babel转换结果，可以看这篇文章https://juejin.cn/post/6844904202502668296#heading-9
