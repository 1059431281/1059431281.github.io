大白话vue响应式原理和双向绑定（记录用，不建议阅读）

这篇文章是给自己用作唤醒记忆的大白话记录，参考价值一一般般

# vue响应式原理

# 问：实现响应式的第一步应该是什么？

答：要能够监听变量的变化！vue2中可以使用Object.defineproperty，缺点是只能监听值类型和对象的第一层，不能监听数组，如果去监听性能上太浪费，vue3使用proxy可以直接解决这些问题

# 问：用监听属性到底要怎么去是现实？

答：我们是在讲最初的数据绑定在data上，而data是vue对象中的一个键，因此我们只要监听这个data对象就好。至于怎么做，就是去遍历这个data对象，判断如果是对象的话就继续去遍历然后进行属性监听，以此不断循环。（vue3的话，proxy直接一步到胃了）

# 问：vue响应式原理采用的模式是什么？

答：观察者模式

# 问：上面提到监听属性这一步是观察者模式的什么？

答：是vue源码中的做监听器observer

# 问：实现响应式的各个模块以及它们之间的关系

答：实现vue的响应式一共由 监听器，观察者（订阅器），负责对相同类型观察者进行管理的dep对象（发布者）

，指令解析器compiler

# 问：还有什么办法快速助于自己回忆？

答：在之前学过已经学过响应式的基础上，看这篇文章可以大致把记忆复苏，只适合有基础的没，没基础的可能会被一些严谨的地方误导https://juejin.cn/post/6989106100582744072#heading-10

# 双向绑定的原理

这个倒是比起响应式简单多了，它就是个语法糖而已，就是将vue中的v-bind和元素本身都onchange组合成v-model，省去我们单向数据操作的麻烦
